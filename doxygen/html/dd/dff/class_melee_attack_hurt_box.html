override void MeleeAttackHurtBox._Ready ( )
inline

Monitoring is set to false so that the collisions from slime and PC only damage the PC if the slime is in the Attacking state. Monitoring is toggled on in the SlimeAttacking state.

◆ OnAreaEntered()

void MeleeAttackHurtBox.OnAreaEntered ( Area2D area)
inline

The slime should only be able to detect collisions with the PC. If the slime collides with the PC, the damage function is called. Afterwards Monitoring is set to false again to prevent the player from taking damage from the same attack a second time. If the slime does not hit the PC, monitoring is turned of in the SlimeAttacking state again.
